# 微信音频二维码 PRD v1.2.2

## 1. 产品定位

提供一个基于 GitHub Pages 的轻量单音频分享方案。用户通过二维码扫码进入播放页，在手机浏览器、微信或桌面浏览器中完成音频收听与下载。

## 2. 产品目标

- 使用免费托管方式完成单音频长期分享
- 支持扫码直达播放页并进行基础播放控制
- 提供可下载的二维码海报，便于社群和线下传播
- 让页面主题、标题、副标题与二维码海报保持统一
- 在移动端场景下提升音频加载、首次播放和下载稳定性
- 在视觉上增强海报页与播放页的一致性和完成度
- 进入播放页后尽量在未点击播放前就显示音频总时长

## 3. 目标用户

- 需要分享单条朗读、班级故事、播客、课程音频的个人用户
- 需要生成长期可复用二维码海报的轻量内容创作者
- 希望使用 GitHub 免费部署完成音频分享验证的个人项目

## 4. 核心使用场景

1. 用户把单条音频上传到公开 GitHub 仓库
2. 用户启用 GitHub Pages 发布播放页和默认音频文件
3. 用户打开二维码生成器，填写标题、副标题、主题等信息
4. 用户下载二维码海报，用于微信、社群、打印海报或活动物料
5. 终端用户扫码进入播放页，点击后播放音频或下载音频文件

## 5. 当前功能范围

### 5.1 播放页 `index.html`

- 采用移动优先的单页播放布局
- 支持主题切换：`aurora`、`podcast`、`story`、`memo`、`radio`、`luxe`、`forest`
- 支持参数化内容：
  - `audio`：音频地址
  - `title`：标题
  - `subtitle`：副标题
  - `theme`：主题
- 支持以下操作：
  - 播放/暂停
  - 快退 10 秒
  - 快进 10 秒
  - 进度拖动
  - 倍速切换 `1 / 1.25 / 1.5 / 2 / 0.75`
  - 下载音频
- 播放页副标题支持以下分段规则：
  - 文本中的换行会拆分为多行
  - 文本中的 `-` 会拆分为多行
- 音频加载策略：
  - 默认使用当前站点下的 `audio.m4a`
  - 当 `audio` 参数传入外部 jsDelivr 音频地址且加载失败时，播放器尝试回退到站内同路径音频
  - 切换音频地址时会主动重新装载音频资源
- 总时长预取策略：
  - 页面初始化后会主动尝试预取当前音频总时长
  - 优先通过独立 `Audio()` 对象读取 `metadata`
  - 若 metadata 未及时返回，再尝试通过 `AudioContext.decodeAudioData()` 解析时长
  - 对默认 `audio.m4a` 提供内置时长提示，作为更稳的首屏兜底
  - 对成功获取过时长的音频进行本地缓存，便于再次进入时优先显示
  - 对默认音频在已有时长提示时跳过重型解码探测，减少与首次真实播放抢占资源
- 下载策略：
  - 点击“下载音频”后优先通过 `fetch` 获取音频内容
  - 获取成功后转为 `blob` 并触发浏览器文件下载
  - 若抓取失败，则回退为直接打开音频链接
  - 下载过程中右上角显示动态“正在下载中...”状态
- 首播优化策略：
  - 移除 `canplay` 阶段的重复 `play()` 调用
  - 首次播放前若检测到 `currentTime` 被异常推进到前几秒，会自动回到 `0`
  - 通过 `playing` 事件记录实际开始播放状态
- 主题视觉策略：
  - 全部主题已同步到更沉稳、更统一的高级感色系
  - `故事朗读` 主题针对班级故事场景做了额外暖色优化
  - `自然清听` 主题针对校园成长与治愈内容做了额外柔和优化
- 已移除：
  - 复制链接按钮和相关功能
  - 播放页底部说明文案

### 5.2 二维码生成器 `qrcode-generator.html`

- 支持输入或配置以下字段：
  - GitHub 用户名
  - 仓库名
  - 分支名
  - 音频文件名
  - 音频直链
  - 标题
  - 海报副标题
  - 播放页副标题
  - 主题
  - 角标文字
- 音频直链留空时，自动拼接 GitHub Pages 同域音频地址
- 支持将标题、播放页副标题、主题和音频地址编码进播放页链接
- 海报副标题与播放页副标题可分别配置
- 海报主题与播放页主题保持一致
- 海报副标题与播放页副标题均支持 `-` / 换行分段规则
- 基于 `QRCode.js` 生成二维码
- 基于 `html2canvas` 导出 PNG 海报
- 支持通过 URL 参数预填表单字段
- 带有状态提示、按钮忙碌状态和库加载校验
- 当前海报主题整体已调整为更沉稳、更有品质感的色彩体系

### 5.3 当前部署方式

- GitHub 仓库需为 `Public`
- 播放页通过 GitHub Pages 发布
- 默认音频通过 GitHub Pages 同域路径访问
- 若用户手动填写外部音频直链，播放页仍可按该直链访问
- 二维码长期可用的前提：仓库公开且页面与音频路径保持稳定

## 7. v1.2.2 主要变化

- 延续 v1.2.1 的总时长预取、内置时长提示与本地缓存逻辑
- 修复播放页“下载音频”按钮在 GitHub Pages / 移动端浏览器中的不稳定问题

## 8. 验收标准

- 公开仓库上传音频和页面文件后，GitHub Pages 可访问
- `audio.m4a` 返回 200 且可在常见浏览器中播放
- 播放页可正常显示标题、副标题、主题和播放状态
- 播放页可完成播放控制和音频下载
- 下载期间可看到动态“正在下载中...”状态
- 副标题中的换行和 `-` 可在播放页与海报页中一致拆分显示
- 生成器可生成二维码并导出 PNG 海报
- 海报主题与播放页主题保持一致
- 海报副标题与播放页副标题可分别配置并分别展示
- `故事朗读` 与 `自然清听` 主题在班级故事场景下具备更明显的适配性

## 9. 已知限制

- 当前仍是单音频分享方案，不支持音频列表或专辑
- 首次播放通常仍需用户手动点击触发
- 不同浏览器对媒体预加载、下载和自动播放策略存在差异，个别环境仍可能出现表现差异
- 海报导出依赖浏览器加载外部 CDN 脚本
- 当前不支持 Logo 上传、JPG 导出、后台管理和统计分析
- 表单中的“分支名”字段会保留显示和预填，但默认 GitHub Pages 音频路径并不依赖该字段

## 10. v1.2.2 交付结论

v1.2.2 已在 v1.2.1 基础上补齐下载音频稳定性、下载过程动态反馈和默认音频首播体验优化，并继续保持总时长预显示、双副标题配置、主题升级和二维码海报生成等能力。当前 PRD 已与代码行为对齐。
