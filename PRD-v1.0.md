# 微信音频二维码 PRD v1.0

## 1. 产品定位

提供一个轻量、低成本、可长期使用的微信音频分享方案。用户通过二维码扫码进入手机端播放页，完成收听、转发和下载。

## 2. 产品目标

- 使用免费方案完成音频托管和分享
- 支持微信内扫码收听
- 页面长期有效，不限制扫码次数
- 提供具备视觉风格的二维码海报输出
- 保持部署简单，适合个人用户快速上手

## 3. 目标用户

- 需要分享单条朗读、播客、课程、语音内容的个人用户
- 需要在朋友圈、社群、名片、海报中传播音频的内容创作者
- 需要低成本方案验证音频传播场景的轻量项目

## 4. 核心场景

1. 用户上传音频到 GitHub 仓库
2. 用户开启 GitHub Pages 作为播放页
3. 用户在本地打开二维码生成器，填写仓库信息
4. 用户下载二维码海报并用于微信或线下场景
5. 终端用户扫码后进入手机端播放页并收听音频

## 5. 功能范围

### 5.1 播放页 `index.html`

- 自适应手机端界面，风格与微信扫码收听场景匹配
- 支持主题切换：`aurora`、`podcast`、`story`、`memo`、`radio`、`luxe`、`forest`
- 支持参数化内容：
  - `audio`：音频地址
  - `title`：标题
  - `subtitle`：副标题
  - `theme`：主题
- 支持以下操作：
  - 播放/暂停
  - 快退 10 秒
  - 快进 10 秒
  - 进度拖动
  - 倍速切换 `1 / 1.25 / 1.5 / 2 / 0.75`
  - 复制当前页面链接
  - 下载音频

### 5.2 二维码生成器 `qrcode-generator.html`

- 输入 GitHub 用户名、仓库名、分支名、音频文件名
- 支持直接粘贴音频直链
- 支持配置海报标题、副标题、主题、角标文案
- 根据输入自动拼接播放页链接
- 基于 QRCode.js 生成二维码
- 基于 html2canvas 导出 PNG 海报
- 支持通过 URL 参数预填生成器字段

### 5.3 部署方式

- GitHub 仓库需为 `Public`
- 音频通过 jsDelivr 加速访问
- 播放页通过 GitHub Pages 发布
- 二维码长期可用的前提：仓库公开且音频路径不变

## 6. 非目标

- 不处理多音频列表和音频专辑系统
- 不提供后台管理系统
- 不提供数据库、登录、统计分析
- 不提供服务端接口

## 7. 文件与职责映射

- `audio.m4a`：示例音频文件
- `index.html`：微信端播放页
- `qrcode-generator.html`：二维码海报生成器
- `deploy.sh` / `立即开始-三步完成.bat`：本地打开生成器
- `README.md` / `🚀快速开始-三分钟完成.md` / `项目说明.txt`：说明文档

## 8. 验收标准

- 公开仓库上传音频和页面文件后，GitHub Pages 可访问
- 音频 CDN 链接返回 200 且可播放
- 扫码或打开播放页后可正常显示标题、副标题、主题
- 播放页可完成播放控制、复制链接和下载音频
- 生成器可输出可扫描的 PNG 海报

## 9. 已知限制

- 首次播放通常需要用户点击触发
- 海报下载依赖浏览器加载外部 CDN 脚本
- 当前版本无 Logo 上传、JPG 下载和批量音频管理能力

## 10. v1.0 交付结论

v1.0 已完成单音频微信扫码收听、播放页展示、二维码海报生成、GitHub 免费部署链路和基础说明文档。
